Introdução

Tendo em conta um aumento significativo da utilização do XML nos últimos anos devido à facilidade com que permite compartilhar informação através da internet, este trabalho tem em vista aplicar os conhecimentos obtidos durante as aulas da cadeira LPROG sobre o tema fornecido "ADOTAPets".

Através do XML, serão organizados e estruturados os dados de uma associação de apoio a animais abandonados, caracterizando todos os elementos dos respetivos animais.

Análise

Neste tema em particular pretende-se desenvolver uma estrutura de dados com vários tipos de informação sobre animais para adoção, como por exemplo nome, raça e informação médica.

A primeira abordagem foi simples, tendo sido feito um pequeno esboço da informação de maneira a apresentar uma ideia geral do problema proposto.

Após uma análise mais aprofundada a solução obtida foi a seguinte estrutura para a constituição do ficheiro:

  • Animais
  • Boxes

Tag "Animais" é constituído por várias estruturas "Animal". E Tag "Boxes" é formada por estruturas "Box", tendo cada Box apenas como atributo, o número da mesma.

A constituição de um animal:

Atributos:

  • TipoAnimal
  • BoxID
  • Chip

Elementos:

  • Nome
  • Raça
  • Idade
  • Sexo
  • Peso
  • Personalidade
  • Fotografia
  • Vacinas
  • Ração
  • ExercicioFisico

O tipo de animal é considerado um enumerado, tendo em conta que só podem ser inseridos três tipos de dados com os seguintes valores:

  • Cão
  • Gato
  • Outro

O box id possui uma referenciação a uma chave, isto é, só podem ser inseridos números de boxes que estejam contidos no conjunto de elementos Boxes. Foi também tido em conta que uma box pode ser utilizada por mais do que um animal.

O atibuto Chip tem um formato fixo para poder ser inserido, de maneira a manter a consistência da informação.

Idade de um animal é obrigatoriamente um número inteiro positivo entre zero e um máximo de 100, isto é, se incluirmos tartarugas.

O sexo de um animal pode apenas ser preenchido como Masculino ou Feminino.

Peso de um animal é constituido por um padrão de no mínimo um digito e um máximo de dois, seguido de no maximo três digitos para as gramas e no finalizado por Kg. Como exemplo: 15.21Kg

A personalidade do animal é tida em conta como qualquer tipo de "string", de forma a cobrir todos os caracteres especiais, essencialmente os acentos.

Vacinas são constituidas por um conjunto de Vacina, sendo por sua vez, cada uma preenchida com o nome, uma data de administração e uma validade.

Ração e Exercicio fisico de um animal é um enumerado de um dos três valores: Normal, Light ou Senior e também Normal, Moderado ou Grande respectivamente.

Linguagem

A nivel de schema, foi assegurado que em tipo de animal apenas se poderia inserir cão, gato ou outro, utilizando-se um enumerado.

Em box ID foi realizada uma referência de id, chaves essas que têm que fazer parte do conjunto de Boxes.

Chip do animal é restrito pelo padrão: PT- seguido de 5 digitos de zero a nove

Nome e raça do animal, constituidos por um padrão que apenas aceita letras e espaços.

As fotografias/imagens não possuem nenhuma restrição em particular a não ser a inserção de um uri para o caminho ou link da fonte.

Cada vacina tem que ser constituida obrigatoriamente por uma data de administração, com um formato fixo do tipo AAAA-MM-DD. A validade da vacina é um enumerado com os valores Mensal, Anual ou Sem Validade.

A ração e exercício físico seguem a mesma estrutura de dados, sendo um enumerado. A ração pode ser Normal, Light ou Senior, e o exercicio físico Normal, Moderado ou Grande.

Transformações

Transformação para HTML 1

Esta transformação consiste em filtrar a informação do ficheiro xml para apresentação de apenas uma tabela com as seguintes informações relativas aos animais:

  • Fotografia
  • Nome
  • Raça
  • Sexo
  • Idade
  • Peso

Transformação para HTML 2

A segunda transformação consiste num plano mais abrangente de informação, apresentando para além dos dados do animal também as respetivas vacinas de cada um, com a data de administração e validade.

Transformação para HTML 3

Esta ultima transformação tem a particularidade de apresentar todos os dados dos animais agrupados por tipo de animal, permitindo uma mais rápida visualização dos dados com interesse.

Transformação para XML 1

Esta transformação tem como atributos as propriedades importantes do animal, tem um elemento nome e um elemento com informação secundária.

Transformação para XML 2

Nesta transformação, associou-se o Animal com o seu nome, a sua box e que vacinas tomou.

Transformação para TXT 1

Esta transformação mostra quantos animais estão para ser adotados e filtra toda a informação pessoal sobre o dado animal.

Transformação para TXT 2

Esta transformação tem como objetivo dar informação médica dos animais a serem adotados, mostrando que vacinas tomou e quando.

Conclusão

Os objetivos do projeto foram alcançados com sucesso, tendo-se conseguido criar um xml validado por um xsd e respetivas transformações para html, texto e xml. O presente relatório foi também validado com o schema fornecido e convertido com sucesso para html.

Ao longo do desenvolvimento do trabalho, foi possível apreciar a utilidade que o xml e xsl apresentam, permitindo guardar e apresentar informação de forma simples e bem organizada. Toda as partes de estilização das páginas html foram inseridas por uma simples questão de brio na apresentação do projeto em questão, mesmo não contando para a valorização do mesmo.
